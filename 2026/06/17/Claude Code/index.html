名词解释

Skills:说明书

Agents:智能体

Hooks:勾子

MCP:模型上下文协议（Model Context Protocol）

Plugins:插件

快捷键

Shift + Tab: 切换模式（计划模式 或者 自动同意编辑模式）

Ctrl + J:换行

Ctrl + G:打开文本编辑器编辑

命令

!:使用shell模式

@:选中文件或者Agent

claude --dangerously-skip-permissions:跳过所有同意（不推荐，非常危险）

/effort:切换思考强度

/exit:退出

/resume:查看最近历史

/init:初始化 CLAUDE.md 文件并写入代码库相关文档（可以把项目规则和技术栈写在里面，每次对话会把这个文件发给模型，文件可能是全英文的，可以使用@CLAUDE.md 帮我翻译成中文的命令来翻译）

/context:查看上下文情况

/compact:压缩上下文空间

/clear:清空上下文

/skills:查看安装的Skill

/agent:管理Agent设置

/plugin:管理 Claude Code 插件

六种权限模式

设置方式如下：

启动的时候通过参数传递，会覆盖 settings 文件里的 defaultMode！

模式 含义 适合场景 风险
default 标准权限模式 日常开发、最稳妥 低
acceptEdits 自动接受文件编辑 写代码、批量改文件 中低
plan 规划模式 大重构、复杂任务、先看方案 低
auto 自动模式 想减少确认，又不想完全放开 中
dontAsk 不询问模式 非交互、自动化、CI、脚本环境 中
bypassPermissions 跳过权限检查 容器、沙盒、临时测试环境 高
