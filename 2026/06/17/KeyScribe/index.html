KeyScribe





模拟真实键盘打字的 Python 脚本 — 从文本文件读取内容，倒计时后自动逐字输入到目标位置。

核心特点：完全不走剪贴板，通过 Windows SendInput API 注入真实键盘事件，不会被识别为粘贴操作。

作者： ol木子李lo · 博客 · GitHub · MIT License

适用场景

  • 网页/应用中禁止了 Ctrl+V 粘贴
  • 远程桌面、虚拟机等无法共享剪贴板的环境
  • 需要模拟真实打字节奏的演示/测试

环境要求

  • Windows 系统
  • Python 3.8 或更高版本
  • 零第三方依赖 — 只使用 Python 标准库

原始模式（Raw Mode）

当你只想原封不动地输入文件内容，不想要 {ENTER} 等特殊按键解析时，使用 -r：

文件中所有字符——包括 {、}、{ENTER} 等——都会被逐字原样打出，不做任何特殊解析。

模式 {ENTER} 效果 { } 效果
正常模式 按下回车键 触发特殊键解析
-r 原始模式 打出 {ENTER} 这几个字符 打出 { } 字符

何时用？ 你的文本文件本身就包含花括号 { } 时，用 -r 避免被误解析。不需要手动逐个加 {{` `}} 转义。

按键中断（默认开启）

这是 KeyScribe 最重要的安全特性。输入过程中，按下键盘上的任意键即可立即中断脚本。

原理：脚本通过 Windows 低级键盘钩子（WH_KEYBOARD_LL）监控键盘事件，利用 LLKHF_INJECTED 标志区分「脚本注入的按键」和「用户真实按键」——只有真实物理按键才会触发中断。

不需要切回终端窗口按 Ctrl+C，只需在当前的输入框中按任意键即可。

关闭此功能： --no-cancel-on-keypress。如果你确定不需要中途打断，可以关闭它。

字符过滤

使用 --filter 选项过滤文本中的特定字符：

模式 效果
no-chinese 移除所有中文字符（包括 CJK 扩展区）
ascii-only 仅保留 ASCII 可打印字符（英文、数字、符号）
no-numbers 移除所有数字

注意： --filter-mode keep 会反转过滤逻辑。如 --filter ascii-only --filter-mode keep 表示仅保留非 ASCII 字符（即中文等）。

常见问题

Q: 中文能正常输入吗？

能。脚本使用 Windows KEYEVENTF_UNICODE 机制注入 Unicode 字符为真实键盘事件。只要目标应用接受键盘输入，就能正常输入中文。

Q: 为什么中文输入不对 / 乱码？

检查文件编码。用 --encoding 指定正确编码：

Q: 会被检测为脚本/自动化吗？

本脚本产生的是 OS 层面的标准键盘事件，和物理键盘没有区别。但过于精确等速的输入可能不自然 — 默认启用的 ±30ms 随机延迟有助于模拟真人打字节奏。

Q: 如何中途取消？

最方便的方式：输入过程中直接按键盘上的任意键（如 ESC、空格等），脚本会立即停止，无需切换回终端。

备选方式：

  • 倒计时阶段 → Ctrl+C 或任意键取消
  • 终端窗口 → Ctrl+C 始终有效
  • 如果不需要按键中断功能，使用 --no-cancel-on-keypress 关闭

Q: 文件很大 (>10000 字符) 怎么办？

脚本会显示警告和预计时间，让你确认是否继续。你也可以随时按 Ctrl+C 中断。

Q: 能在 macOS / Linux 上用吗？

目前仅支持 Windows。macOS/Linux 需要不同的实现方式（如 Quartz 或 uinput），后续可扩展。

Q: 有哪些参数？怎么查看帮助？

运行 python keyscribe.py --help，会以中英双语显示所有参数、说明和使用示例。
